Detailed Description

Represents a terrain.

This class uses different LoDs (quad tree) and tessellation to render the terrain efficiently with high detail without sacrificing too much on performance.

Constructs a Terrain object.

Parameters
rootNodeSideCountThe number of root nodes per side. Root nodes are the lowest level of detail.
LoDCountThe number of level of detail levels. Results in LoDCount - 1 subdivisions of the root nodes.
patchSizeFactorChanges the size of a patch and the number of vertices in a patch
resolutionLike a scale in x and z direction of the terrain. Lower means more vertices per square unit.
heightThe maximum height of the terrain
Note
The maximum number of nodes in a terrain is 2^16. A node has 8 * 8 = 64 patches. The number of vertices per patch corresponds to 4 * pow(2 * patchSizeFactor, 2).
Remarks
Let's assume we have a rootNodeSideCount of 3 which leads to rootNodeCount = 3^2 = 9. We now decide that we want an LoDCount of 7. To check if we don't exceed the maximum number of nodes we calculate rootNodeCount * (pow(4,LoDCount) - 1) / 3 = 3 * (pow(4, 7) - 1) = 49149 < 2^16 = 65536. This means that at the maximum lod which is Lod6 we have a total of pow(4, 6) * 9 = 36864 nodes. If we have a patchSizeFactor of 4 we have 4 * pow(2 * 4, 2) = 64 vertices per patch. This results in a maximum vertex count of around 600 mio. If we take the square root we get the number of vertices per side: sqrt(603979776) = 24576. If we set the resolution to 0.5 we get a total of 12288 units per terrain side. The size of map is therefore roughly 150 kilounits^2.
